### ``Multipart Upload

O **Multipart Upload** é um recurso do Amazon S3 que permite fazer upload de objetos grandes ==dividindo-os em partes menores==, proporcionando:

- **Uploads mais rápidos** através de upload paralelo
    
- **Melhor recuperação de falhas** (apenas partes falhas são retransmitidas)
    
- **Uploads de objetos muito grandes** (até 5TB)
    
- **Uploads resumíveis**

### ``S3 Transfer Acceleration

O **S3 Transfer Acceleration** é um recurso do Amazon S3 que otimiza a velocidade de transferência de arquivos para buckets localizados em regiões distantes do usuário, utilizando a rede global da AWS ==CloudFront.==

# Upload tradicional (rota direta)
Cliente → Internet → Região S3 (possível rota subótima)
# Com Transfer Acceleration
Cliente → Edge Location mais próxima → AWS Backbone → Região S3

 
**S3 Standard-IA: O **S3 Standard-Infrequent Access (Standard-IA)** é projetado para dados acessados com pouca frequência, mas que exigem **acesso rápido e** quando necessário. Ele oferece um custo de armazenamento significativamente menor que o S3 Standard, tornando-o ideal para backups antigos que raramente são acessados, mas onde a recuperação rápida ainda é valiosa.

## ==**Situação:**== 
Uma empresa possui um bucket do Amazon S3 que contém dados críticos. A empresa precisa proteger os dados contra exclusão acidental.

**Controle de Versão:** Quando habilitado, o controle de versão do S3 mantém todas as versões de um objeto. Se uma versão "atual" de um objeto for excluída acidentalmente, ela se torna uma **exclusão lógica** (é marcada como um "marcador de exclusão"), mas a versão anterior do objeto ainda é preservada e pode ser restaurada. Esta é a defesa fundamental contra exclusão acidental.

**Exclusão de MFA:** A Exclusão de MFA (Autenticação Multifator) adiciona uma camada extra de segurança para **excluir permanentemente uma versão de objeto**. Para excluir uma versão de objeto permanentemente (tornando-a irrecuperável), o usuário deve fornecer o código serial do dispositivo MFA e uma código de autenticação de seis dígitos. Isso impede que exclusões acidentais ou mal-intencionadas se tornem permanentes.

### ==**S3 Object Lock==

O **S3 Object Lock no modo de conformidade** impede que um objeto seja excluído ou sobrescrito por _qualquer usuário_, incluindo o usuário root da conta, pelo período de retenção definido. No modo de conformidade, esse período **não pode ser encurtado**.

A permissão `s3:PutObjectLegalHold` pode ser concedida apenas aos usuários específicos autorizados a **remover** a retenção legal (desbloqueando o objeto para exclusão).

--------------------------------------------------------------------------

### ==**S3 Standard-IA== (Acesso Infrequente Padrão)**

Oferece **acesso imediato** (baixa latência em milissegundos) aos dados, com um custo de armazenamento mais baixo que o S3 Standard, mas com uma taxa de recuperação por GB.

### **Origem Access Control (OAC)**

Para restringir o acesso a um bucket do S3, permitindo-o **somente através do CloudFront** e bloqueando o acesso direto, o padrão é usar uma **Origem Access Identity (OAI)** ou sua versão mais recente, a **Origem Access Control (OAC)**.

- A **OAI** é uma identidade virtual associada à sua distribuição do CloudFront.
    
- Você cria uma política de bucket no S3 que **concede permissões de leitura (`s3:GetObject`) apenas a essa OAI específica**.
    
- Como resultado, qualquer tentativa de acessar os objetos diretamente pela URL do S3 (sem passar pelo CloudFront) será negada, pois a identidade do solicitante não será a OAI.
